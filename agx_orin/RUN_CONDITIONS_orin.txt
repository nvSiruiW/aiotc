=== STEP 6 INT8/TensorRT (completed with adaptation) ===
TensorRT: 11.1.0.106+cuda13.3 (apt, sbsa repo). trtexec at /usr/bin/trtexec.
export_trt.py did NOT run on TRT 11.1: it uses IInt8EntropyCalibrator2 + BuilderFlag.FP16/INT8,

Adapted pipeline (scripts/export_trt_trt11.py): precision now lives in the ONNX graph:
  FP16 = float16 ONNX (onnxconverter_common); INT8 = QDQ ONNX from onnxruntime static
  quantization over the REAL calib windows, forced symmetric (TRT needs zero_point==0)
  and QuantizeBias=False (TRT rejects Int32 bias QDQ). TRT then builds strongly-typed.

Engines: engines/orin/{model}_{fp16,int8}.plan (6 CNNs x 2). Latency via trtexec.
FINDINGS: FP16 ATE within +-0.2% of FP32 (device-independent). INT8 ATE +35%..+82% WORSE
than FP32 on every model (measured, NOT assumed) -> INT8 PTQ is not accuracy-free for this
velocity-regression task. TRT engines ~20-40x faster than the PyTorch-eager profile;
INT8-vs-FP16 latency is mixed (Q/DQ overhead offsets gains on the smallest models).
NOTE: INT8 calibration = ORT MinMax symmetric PTQ; entropy/percentile calibration or QAT
might recover some accuracy but was not pursued.
